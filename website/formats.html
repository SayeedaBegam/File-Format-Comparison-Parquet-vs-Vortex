Parquet vs Vortex

High‑level differences in layout, encoding, and typical access patterns.

Parquet

Columnar, file‑based format with row groups and per‑column encodings.

Optimized for scan‑heavy analytics and predicate pushdown.

Widely supported across query engines.

Vortex (DuckDB)

Columnar format integrated with DuckDB’s Vortex extension.

Designed for high‑performance scans and fast decompression.

Best used inside DuckDB where the extension is available.

How the benchmark compares them

Both formats are written from the same in‑memory table and scanned using DuckDB.

Write path

Parquet: COPY (SELECT * FROM t) TO 'file.parquet' (FORMAT PARQUET)

Vortex: COPY (SELECT * FROM t) TO 'file.vortex' (FORMAT VORTEX)

Read path

Parquet: read_parquet('file.parquet')

Vortex: read_vortex('file.vortex')

When each shines

Use the recommendations panel in reports to choose based on your goals.

Parquet is strong when…

You need interoperability, established tooling, and stable long‑term storage.

Vortex is strong when…

Your workflow runs in DuckDB and you want the best scan performance within that engine.
