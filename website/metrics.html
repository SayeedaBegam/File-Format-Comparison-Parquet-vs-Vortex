Metrics overview

Quick read of the signals we track and how to interpret them.

Storage efficiency

  • compression_ratio -- higher is better.
  • compressed_size -- lower is better.
  • compression_time_s -- lower is better.

Query latency

  • median_ms -- typical latency.
  • p95_ms -- tail latency.
  • cold_ms -- first-run latency.

Selectivity behavior

  • selectivity curve -- slope shows pruning.
  • random_access -- point reads.
  • LIKE predicates -- text scans.

Reliability

  • runs -- more runs = stable.
  • variance -- lower is stable.
  • validation -- correctness check.

Query definitions

Use-case framing for the SQL patterns we time across formats.

Full scan
min(min_col)
Read the entire column to compute the minimum.
Selective filter
filter_col = filter_val
Single filter to test pruning behavior.
Random access
point lookup on high-NDV column
Small target reads and seek overhead.
Selectivity curve
min(min_col) at 1/10/25/50/90%
Performance slope as selectivity increases.
LIKE predicates
prefix / suffix / contains
Pattern scans over text columns.

Diagnostics

Small checks to explain why formats behave differently.

Cold vs warm

Cold run is measured once before warmup. Warm timing is the median of repeated runs.

Cache state

Cold runs avoid cache effects; warm runs show steady-state performance.

Row group behavior

Sorted columns enable pruning; unsorted scans more blocks.

Format notes

Missing extensions or errors appear next to the format.
