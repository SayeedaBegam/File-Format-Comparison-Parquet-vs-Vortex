Metrics overview

Quick read of the signals we track and how to interpret them.

Storage efficiency

  • compression_ratio — input_size / output_size (higher = smaller files).
  • compressed_size — final file size on disk (lower = better).
  • compression_time_s — time to write the file (lower = faster writes).

Query latency

  • median_ms — median over timed runs (steady-state).
  • p95_ms — tail latency over timed runs.
  • cold_ms — first run before warmup (cache-cold).
  • warmup — warmup runs before timing (default 1).
  • runs — timed runs used for median/p95 (default 7).

Selectivity behavior

  • selectivity curve — latency vs selectivity thresholds.
  • random_access — point lookup latency on high‑NDV column.
  • LIKE predicates — text pattern scans (prefix/suffix/contains).

Reliability

  • runs — more runs reduce noise.
  • variance — lower = more stable.
  • validation — base table vs format correctness checks.

Query definitions

Use-case framing for the SQL patterns we time across formats.

Full scan
min(min_col)
Read the entire column to compute the minimum.
Selective filter
filter_col = filter_val
Single filter to test pruning behavior.
Random access
point lookup on high-NDV column
Small target reads and seek overhead.
Selectivity curve
min(min_col) at 1/10/25/50/90%
Performance slope as selectivity increases.
LIKE predicates
prefix / suffix / contains
Pattern scans over text columns.

Diagnostics

Small checks to explain why formats behave differently.

Cold vs warm

Cold is a single run before warmup. Warm uses median/p95 from timed runs after warmup.

Cache state

Cold reflects cache-miss behavior; warm reflects steady-state scans.

Format notes

Any missing extensions or runtime errors are shown next to the format.
